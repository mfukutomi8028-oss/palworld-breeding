# パルワールド配合記録ボード v7

GitHub Pagesで公開できる、Firebase Realtime Database対応の配合記録ボードです。

## v7 修正内容

- 左上・メニューのタマゴ画像を、ユーザー指定の「平凡なタマゴ」画像に差し替えました。
- ブラウザタブのfaviconを、ユーザー指定の公式風パル画像に設定しました。
- ネタバレ防止のため、未登録パルの画像は「？」画像で表示するようにしました。
- 一度でも配合記録に追加されたパルだけ、一覧・詳細・入力候補・絞り込み候補で実画像が表示されます。
- パル名はこれまで通り表示されます。画像のみ伏せます。

## GitHub Pagesへの反映

zipを解凍し、中身をリポジトリ直下へ上書きアップロードしてください。

- index.html
- style.css
- app.js
- config.js
- config.example.js
- firebase-rules.json
- README.md
- assets フォルダ

反映後、公開ページで `Ctrl + F5` を押してください。
古い表示が残る場合は、URL末尾に `?v=7` を付けて確認してください。

## Firebaseについて

`config.js` にFirebase設定を入れると共同編集ONになります。
未設定の場合はブラウザ内のローカル保存で動きます。


## v8 修正内容

- フブキジカの画像が表示されない問題を修正しました。
- 未登録パルの画像は、追加フォーム・絞り込み候補・一覧で「？」画像になるよう再調整しました。
- 一度でも配合記録に登場したパルのみ画像を表示します。
- 登録画面の「確認状況」入力を廃止しました。
- 結果パルが入力されている場合は自動で「配合確認済み」、未入力の場合は自動で「確認中」になります。
- タブアイコンを `favicon-v8.png` に変更し、ブラウザキャッシュを避けるためURLに `?v=8` を付けています。
- 平凡なタマゴ画像を添付画像から切り抜き、`assets/plain-egg.png` として同梱しました。


## v10 修正内容

v9でひらがな検索処理が実ファイルに反映されていなかったため、正規化処理を再修正しました。

- 親A / 親B / 結果パルの候補検索で、ひらがな入力に対応
- 左メニューの親パル / 結果パル絞り込みで、ひらがな入力に対応
- 中央の検索欄で、ひらがな入力に対応
- ひらがな入力のまま保存した場合、パル一覧にある名前はカタカナへ正規化
- `app.js?v=10` にしてブラウザキャッシュを回避
- パル一覧キャッシュキーをv10に更新

例：
- もころん → モコロン
- ふぶきじか → フブキジカ
- らべろっと → ラベロット
- ぺこどん → ペコドン


## v11 修正内容

- 同じ親A・親Bの組み合わせが既に登録済みの場合、新規追加できないようにしました。
- 親Aと親Bが逆でも同じ組み合わせとして判定します。
  - 例：`モコロン + ツッパニャン` と `ツッパニャン + モコロン` は同一扱いです。
- 編集中の同一レコードは重複扱いにせず、そのまま更新できます。
- `app.js?v=11` にしてブラウザキャッシュを回避しています。


## v12 修正内容

- 配合記録に「タマゴの種類」欄を追加しました。
- タマゴ欄はパル選択と同じように、入力しながら候補選択できます。
- 一覧・詳細・候補にタマゴ画像を表示します。
- タマゴ画像は添付画像を `assets/eggs/` に同梱しています。
- 対応タマゴ：
  - 平凡なタマゴ
  - 熱を帯びたタマゴ
  - しめったタマゴ
  - 新緑のタマゴ
  - ビリビリのタマゴ
  - ゴツゴツしたタマゴ
  - 凍てつくタマゴ
  - 暗黒タマゴ
  - 竜のタマゴ
- 重複登録時のエラーを、登録ダイアログ内にも表示するようにしました。
- `app.js?v=12` にしてブラウザキャッシュを回避しています。


## v13 修正内容

- タマゴの種類に「デカ」「キョダイ」を追加しました。

- 9種類 × 通常/デカ/キョダイ の合計27種類から選択できます。
